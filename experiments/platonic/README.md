# Platonic-Universe paper reproduction

Two reproduction tiers, opt-in by `pip install` extras. Figures are
intentionally not included yet — they will be added once the figure list
for the paper is finalised.

| Tier | Inputs | Output | Hardware | Extra |
|---|---|---|---|---|
| `compute/` | layer-wise embeddings on a public HF dataset | calibrated CKA + MKNN + UMAP parquets | 1+ A100 GPU | `platonic-compute` |
| `extract/` | raw galaxy images + foundation-model checkpoints | the layer-wise embeddings that `compute/` consumes | many GPU-hours per model | `platonic-extract` |

External dependencies that **cannot** be regenerated by this repo:
- The crossmatched HSC pairing datasets at `Smith42/<survey>_hsc_crossmatched`.
- The COSMOS-Web HSC×JWST sample at `Ashodkh/cosmosweb-hsc-jwst-high-snr-pil2`.
- Foundation-model checkpoints on Hugging Face (subject to upstream changes).

Data inputs (layer-wise embeddings, the per-(model, survey) parquets, the
`candidates.parquet` consumed by Phase 2) all live on Hugging Face datasets
that the reader points at via env vars (`PU_*_EMBED_REPO`, `PU_*_RESULTS_REPO`).
**Nothing data-shaped is committed to git** — this repo carries only the code
that produces and consumes those parquets.

## Layout

```
experiments/platonic/
├── compute/
│   ├── pu_solve.py             # phase 1: per-pair calibrated CKA + MKNN + per-block shape stats
│   ├── pu_umap.py              # phase 2: kNN-purity + per-row TwoNN + UMAP per candidate
│   ├── select_candidates.py    # picks top-3 HSC blocks per (model, survey) for phase 2
│   ├── merge_partials.py       # combines pair-range partition outputs from pu_solve.py
│   ├── setup_solve_hf.py       # one-time bootstrap of the phase-1 coordination dataset
│   ├── setup_umap_hf.py        # one-time bootstrap of the phase-2 coordination dataset
│   ├── slurm/                  # SBATCH wrappers for SGE-style clusters
│   └── vast/                   # tmux launchers for vast.ai multi-GPU rentals
└── extract/
    └── extract_layerwise.py    # batch wrapper around pu's layerwise CLI
```

## Quick start (compute tier)

```bash
pip install -e ".[platonic-compute]"
export HF_TOKEN=...                                     # required, no default
export PU_SOLVE_RESULTS_REPO=<owner>/pu-solve-results   # your own coordination dataset
export PU_SOLVE_EMBED_REPO=<owner>/platonic-embeddings  # source of the embeddings
python experiments/platonic/compute/setup_solve_hf.py "$PU_SOLVE_RESULTS_REPO"
python experiments/platonic/compute/pu_solve.py
```

After it finishes (hours per tuple, 144 tuples total):

```bash
python experiments/platonic/compute/select_candidates.py
export PU_UMAP_RESULTS_REPO=<owner>/pu-umap-results
export PU_UMAP_EMBED_REPO="$PU_SOLVE_EMBED_REPO"
python experiments/platonic/compute/setup_umap_hf.py "$PU_UMAP_RESULTS_REPO"
python experiments/platonic/compute/pu_umap.py
```

## Notes

- The compute pipeline is designed to be run by many workers concurrently, on
  any number of clusters, coordinating via a private Hugging Face dataset.
  Workers atomically claim tasks via `running/<tag>.running` markers; stale
  claims (>1 hour) are reaped automatically.
- For the heavy tail (paligemma-28B on legacysurvey, $N \sim 10^5$), the
  `compute/vast/vast_paligemma.sh` launcher splits the $\sim 5400$-pair grid
  across 8 GPUs; `merge_partials.py` consolidates the outputs.
- All inputs and outputs are parquets. No databases, no central scheduler.
